# Objective
NextAuth.js を導入し、`@cuebic.co.jp` のGoogleアカウントを持つユーザーのみがアプリにアクセスできるようにアクセス制限（Middleware）をかける。

# Execution Steps (Cursor Composer)
以下のステップで既存のコードを修正してください。

1. [Setup]
   - ターミナルで `npm install next-auth` を実行する。

2. [NextAuth Configuration]
   - `app/api/auth/[...nextauth]/route.ts` を作成し、GoogleProviderを設定する。
   - `callbacks: { signIn }` を使用し、`profile.email.endsWith('@cuebic.co.jp')` の場合のみログインを許可するロジックを実装する。弾かれた場合はデフォルトのエラーページで構わない。

3. [Middleware (Route Protection)]
   - プロジェクトのルート（`app/` と同じ階層）に `middleware.ts` を作成する。
   - `withAuth` を使用し、未ログインユーザーがアクセスしてきた場合は自動的にGoogleログイン画面（NextAuthのデフォルトログイン画面）にリダイレクトさせる。すべてのルート（`/` 以下のすべて）を保護対象とする。

4. [UI Update - 任意]
   - アプリケーション全体でセッション情報を取得できるよう、`app/layout.tsx` または別のClient Componentにて `SessionProvider` をラップする設定を行う（App Routerの仕様に従うこと）。
   - ヘッダー部分などに、現在ログインしているユーザーのアイコンやメールアドレス、および「ログアウト」ボタンを表示する。

# Constraints
- 型定義（TypeScript）を厳格に保ち、コンパイルエラーを出さないこと。
- 余計な説明は省き、コード生成・修正に集中すること。
